## 【问题陈述】幻灯片 2

### 标题：现实痛点——数据汇总的"三座大山"

### 核心要点
1. **数据散乱**：关键数据分散在数十个 Excel、Word、PDF 文件中
2. **格式异构**：不同来源文档格式迥异，无法自动对接
3. **人工填写**：分析人员耗费大量时间做重复性"复制-粘贴"

### 数字冲击
- 一份中等规模报表，人工处理平均耗时：**2～4 小时**
- 人工差错率：约 **3%～8%**
- 企业每年因此损失的生产效率：不可忽视

> 引导观众思考：有没有更好的方法？→ 引出解决方案

## 【解决方案】幻灯片 3

### 标题：LLM-DocFusion —— 全流程自动化解决方案

### 核心流程图（文字版）

```
多源文档输入
  Excel / Word / Markdown / TXT / CSV
        ↓
  [统一文档解析]  →  DocumentBundle 中间表示
        ↓
  [需求理解]      →  时间范围 · 关键实体 · 过滤条件
        ↓
  [证据检索]      →  相关文本块 · 候选表格
        ↓
  [智能数据抽取]  →  规则匹配 + LLM 辅助
        ↓
  [模板自动填充]  →  写入目标格式，保留原始样式
        ↓
  [结果验证]      →  填充率 · 证据追溯 · 异常告警
        ↓
 标准化报表输出（一键下载）
```

### 核心价值
- ⏱ **效率提升**：2～4小时 → **当前样例约 74～86 秒**
- 📊 **输出填充率**：当前已验证样例 **89.3%～96.5%**
- 🔒 **数据安全**：完全本地推理，数据不出服务器

## 【技术架构】幻灯片 4

### 标题：系统架构——分层解耦，模块可扩展

### 架构分层说明

| 层次 | 组件 | 说明 |
|:----:|:----:|:----:|
| 展示层 | HTML / CSS / JS | 拖拽上传、实时进度条、结果预览与下载 |
| 服务层 | FastAPI REST API | 异步任务调度、文件管理、状态查询 |
| 处理层 | 7大服务模块 | 解析→检索→抽取→填充→验证全流程 |
| 推理层 | Ollama + qwen2.5:14b | 本地 LLM 推理，无需云端 API |

### 技术亮点
- **异步任务管理**：支持并发处理多个任务，实时推送进度
- **DocumentBundle 中间层**：统一多格式表示，彻底解耦解析与处理
- **混合抽取策略**：规则优先（快、稳）+ LLM fallback（智能、灵活）

## 【核心创新】幻灯片 5

### 标题：三大核心创新点

### 创新一：DocumentBundle 统一中间表示
- 原创设计的文档中间表示层
- 将 5 种格式的文档统一归一化
- 后续所有算法模块完全格式无感知
- **效果**：新增格式支持只需修改解析器，零改动下游

### 创新二：规则优先 + LLM 增强混合抽取引擎
- 高置信度字段优先使用确定性规则
- 歧义/跨段落场景自动切换 LLM 辅助
- 多实体场景（百强榜等）分块迭代，不遗漏任何实体
- **效果**：兼顾速度与覆盖率，无需人工干预

### 创新三：置信度驱动的自动验证与证据追溯
- 每个填充字段均计算置信度分数
- 自动生成数据来源追溯报告
- 填充率、空白字段、异常值三维验证
- **效果**：结果可信、可查、可审计

## 【技术栈】幻灯片 8

### 标题：技术栈全景

### 核心技术栈

```
后端框架    │  FastAPI 0.110+  │  高性能异步 API
LLM 推理   │  Ollama          │  本地化大模型推理框架
大语言模型  │  qwen2.5:14b    │  通义千问（140亿参数）
文档处理   │  openpyxl        │  Excel 读写
           │  python-docx     │  Word 读写
数据处理   │  pandas          │  表格数据处理
语言运行时  │  Python 3.10+    │
前端        │  原生 HTML/CSS/JS │  轻量级，零依赖
部署        │  单机启动脚本     │  ./start.sh 一键启动
```

### 选型理由
- **Ollama + qwen2.5:14b**：国产最优中文大模型之一，本地部署，无 API 费用
- **FastAPI**：原生 async 支持，适合大文件上传与长任务处理
- **无外部 AI 服务依赖**：100% 自主可控，适合政务/企业私有化部署
