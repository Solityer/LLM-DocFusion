| 项目名称 | LLM-DocFusion：基于大语言模型的多源文档智能融合与模板自动填充系统 |
|:-------:|:-----:|
| 项目类别 | 人工智能 · 文档智能处理 |

---

## 二、项目背景与动机

在政府机关、科研单位、金融机构、企业管理等实际业务场景中，**数据报告的撰写与汇总**是一项极为繁琐且耗时的工作。数据分析人员往往需要：

1. 从多个来源文件（Excel 统计表、Word 报告、PDF 政务公报、数据库导出等）中人工检索关键数据；
2. 将检索到的数据手动对应填入规定格式的模板表格；
3. 反复核查数据准确性，防止因格式不匹配或笔误导致的错误。

这一流程不仅效率低下，还存在**数据来源追溯困难**、**人工填写差错率高**等问题。

**LLM-DocFusion** 正是为解决上述痛点而生。系统借助本地部署的大语言模型，实现了从"多源文档读取 → 需求理解 → 证据检索 → 智能抽取 → 模板自动填充 → 结果验证"的**全流程自动化**，显著降低人力成本，提升数据处理效率与准确率。

---

## 三、项目核心定位

> **让 AI 替代人工，完成从"散乱数据"到"标准报表"的全自动转换。**

**三个"多"与"自动"**：
- **多源**：支持 Excel、Word、Markdown、TXT、CSV 等多种格式同时输入；
- **多模板**：任意格式的文件均可作为填充目标模板；
- **多实体**：自动识别并提取百强榜、排名表等多实体场景数据；
- **自动化**：从上传文件到输出结果，无需人工干预，全流程自动完成。

## 四、核心技术能力

### 4.1 多源异构文档统一解析

| 格式 | 解析方式 | 特性 |
|:----:|:-------:|:----:|
| Excel (.xlsx/.xls) | openpyxl | 保留合并单元格信息、多 Sheet 支持 |
| Word (.docx) | python-docx | 段落 + 表格双通道提取 |
| Markdown (.md) | 自研解析器 | 标题层级、表格、列表结构化 |
| TXT (.txt) | 规则分块 | 段落切分、关键词定位 |
| CSV (.csv) | pandas | 表头对齐、数据类型推断 |

所有格式统一归一化为 **DocumentBundle** 中间表示，后续处理完全无感知原始格式。

### 4.2 规则优先 + LLM 增强的混合抽取策略

```
证据检索 → 规则匹配（高置信度）
              ↓（规则不足时）
           LLM 辅助抽取（qwen2.5:14b）
              ↓（多实体场景）
           分块迭代提取（chunk-based multi-call）
              ↓
           结果去重合并
```

- 规则匹配优先保证效率与确定性；
- LLM fallback 处理歧义、跨段落、非标准表达；
- 多实体场景采用分块多次调用，确保百强排名等大量实体不遗漏。

### 4.3 全格式模板填充与结果验证

- 读取模板结构（表头、占位符、可写区域、合并策略）；
- 按行写入抽取数据，**保留原始格式、样式、公式**；
- 自动计算每列置信度，并生成 **数据来源追溯报告**；
- 验证填充完整性：填充率、空白字段告警、异常值检测。

## 六、关键指标与优势

| 指标 | 数值 |
|:----:|:----:|
| 支持输入格式数 | 5 种（Excel/Word/MD/TXT/CSV） |
| 支持模板格式数 | 5 种（同上） |
| 当前已验证样例填充率 | 89.3%～96.5%（见 `outputs/api_verification_final.json`） |
| 当前本地验证耗时（多模板） | 74～86 秒（依赖数据规模与 LLM 调用次数） |
| 数据隐私保护 | 全本地推理，无外传 |
| 系统部署方式 | 一键启动（`./start.sh`） |
| LLM 模型 | qwen2.5:14b（通义千问 2.5，140 亿参数） |

**对比传统方案优势：**

| 维度 | 人工填写 | 传统 RPA | LLM-DocFusion |
|:----:|:-------:|:--------:|:-------------:|
| 适应新模板 | 需人工 | 需重新配置 | **开箱即用** |
| 跨格式支持 | 低 | 低 | **高** |
| 理解自然语言需求 | 人工理解 | 不支持 | **LLM 自动理解** |
| 数据隐私 | 安全 | 依赖配置 | **全本地，安全** |
| 部署成本 | 无 | 高 | **低（单机）** |

## 七、应用场景

1. **政务数据汇总**：将多省/市/县统计公报中的指标数据自动汇总填入标准化报告模板；
2. **科研数据整理**：从多篇论文/数据集中提取实验数据，自动填入对比分析表；
3. **金融报表生成**：从多个业务系统导出数据，自动填入财务报告模板；
4. **医疗卫生统计**：将各地区卫生统计数据汇总为全国性报表；
5. **企业竞争情报**：从行业报告中提取竞品数据，自动更新竞争分析矩阵。
