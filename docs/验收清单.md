## 五、主填表流程演示

### 5.1 使用 curl（推荐快速验证）

```bash
# 上传数据源
SRC=$(curl -s -X POST http://localhost:8000/api/files/upload \
  -F "files=@测试集/包含模板文件/COVID-19数据集/COVID-19全球数据集（节选）.xlsx" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['files'][0]['path'])")
echo "数据源路径: $SRC"

# 上传模板
TPL=$(curl -s -X POST http://localhost:8000/api/templates/upload \
  -F "files=@测试集/包含模板文件/COVID-19数据集/COVID-19 模板.xlsx" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['files'][0]['path'])")
echo "模板路径: $TPL"

# 创建处理任务
TASK=$(curl -s -X POST http://localhost:8000/api/process/local \
  -H 'Content-Type: application/json' \
  -d "{\"source_files\":[\"$SRC\"],\"template_files\":[\"$TPL\"],\"requirement\":\"\",\"options\":{\"use_llm\":false}}" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['task_id'])")
echo "任务ID: $TASK"

# 轮询直到完成
while true; do
  STATUS=$(curl -s http://localhost:8000/api/status/$TASK)
  STATE=$(echo $STATUS | python3 -c "import sys,json; print(json.load(sys.stdin).get('status',''))")
  echo "状态: $STATE"
  [ "$STATE" = "completed" ] || [ "$STATE" = "error" ] && break
  sleep 2
done

# 查看结果
curl -s http://localhost:8000/api/result/$TASK | python3 -m json.tool | head -40

# 查看报告
curl -s http://localhost:8000/api/report/$TASK | python3 -m json.tool | head -30

# 下载输出文件
OUTFILE=$(curl -s http://localhost:8000/api/result/$TASK \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('results',[{}])[0].get('output_file','') if d.get('results') else '')" \
  | xargs -I{} basename {})
echo "输出文件: $OUTFILE"
curl -s -O http://localhost:8000/api/download/$OUTFILE
ls -la $OUTFILE
```

## 十一、常见问题与处理

### Q1: Ollama 不可用 / ollama_status=error

**现象：** `GET /api/health` 返回 `ollama_status: "error"` 或 `"offline"`

**影响：** LLM 增强功能降级，使用规则引擎提取数据。填充率可能下降但系统不崩溃。

**处理：**
```bash
# 检查 Ollama 服务
curl http://localhost:11434/api/tags
# 如未运行，启动：
ollama serve &
# 拉取模型：
ollama pull qwen2.5:14b
```

**验收底线：** 即使 Ollama 不可用，主流程、存储、分析等功能均可正常演示。

---

### Q2: 外部网页/API 访问失败

**现象：** `POST /api/sources/preview` 返回 `status: "partial"` 和 errors

**处理：** 这是 **fail-soft 设计**，是预期行为。本地文件和处理流程不受影响。

```bash
# 验证 fail-soft：
curl -s -X POST http://localhost:8000/api/sources/preview \
  -H 'Content-Type: application/json' \
  -d '{"source":{"source_type":"http_api","name":"bad","url":"http://localhost:19999/bad"}}'
# 应返回 status=partial，errors 列表，不返回 500
```

---

### Q3: SQLite 文件不存在

**现象：** 首次运行 `/api/store/stats` 时数据库不存在

**处理：** 系统自动懒初始化。调用任意 store API 会自动创建 `data/docfusion_store.sqlite`。

```bash
ls -la data/docfusion_store.sqlite
# 如果不存在，调用一次：
curl http://localhost:8000/api/store/stats
# 然后再查看
```

---

### Q4: 上传文件格式不支持

**支持的格式：** `.xlsx`, `.xls`, `.csv`, `.docx`, `.md`, `.txt`, `.json`

**不支持：** `.pdf`, `.pptx`, `.zip`, `.doc`（旧版 Word）

**处理：** 前端不限制文件类型，后端读取失败会在 `warnings` 中说明，不崩溃。

---

### Q5: 任务长时间 processing 不结束

**原因：** LLM 生成慢（大文件 + 复杂字段），或 Ollama GPU 资源不足。

**处理：**
```bash
# 使用 use_llm=false 跳过 LLM（规则模式更快）
# 在 API 调用中加入：
"options": {"use_llm": false}

# 查看实时日志：
tail -f logs/docfusion.log
```

如果任务卡在 `processing` 超过 5 分钟，可重启服务（任务在内存中，重启后会丢失，但文件不丢失）。

## 十二、已知限制

1. **Ollama 不可用时**：摘要/提取的 `use_llm=true` 请求会降级到规则路径，准确率可能下降。这是正常行为。

2. **extract_entities 的实体识别**：规则引擎只能识别日期、数字、百分比等模式化实体。复杂命名实体（人名、机构名）需要 LLM，LLM 不可用时返回空列表。

3. **evaluate/compare 文件格式**：只支持 Excel/CSV 的准确率对比。DOCX/TXT 会返回 HTTP 400 说明不支持。

4. **analytics avg_fill_rate 显示**：当值为 0.0（无任务数据）时，前端显示"-"而非"0.0%"（0 在 JS 中为 falsy，这是预期行为）。

5. **Web 页面抓取**：`web_page` 数据源依赖公网访问，本地测试环境可能因网络限制失败。这是 fail-soft 设计，不影响本地文件流程。

6. **SQLite 外部数据源**：`database` source 类型需要提供服务器本地绝对路径，前端有文字说明。

7. **time_budget_seconds**：后端接收此参数并用于 LLM 超时降级判断，但不保证硬性截断。

8. **store import 覆盖**：相同文件名重复入库时，如不加 `overwrite=true`，会跳过（`status=completed, 跳过 1`）——这是 SHA256 去重的正常行为。
