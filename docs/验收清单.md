## 十、已知限制

1. **Ollama 不可用时**：LLM 相关功能（summarize/extract with LLM）会降级到规则路径，字段提取准确率会下降。此为正常行为，前端显示"model not used"。

2. **非结构化文本的 extract_entities**：规则引擎仅能识别日期、数字、百分比等模式化实体。复杂命名实体（如人名、机构名）需要 LLM 支持，LLM 不可用时返回空列表，这是正常行为而非 bug。

3. **evaluate/compare 的 accuracy**：只计算可以用 pandas 读取的 Excel/CSV 文件。其他格式（docx、txt）不支持对比，会返回 HTTP 400。

4. **analytics 的 avg_fill_rate 显示**：当 avg_fill_rate = 0.0（无任务数据）时前端显示"-"而非"0.0%"，这是预期行为（0 在 JS 中为 falsy）。

5. **Web 页面抓取**：`web_page` source 依赖公网访问，本地测试环境可能因网络限制导致 preview 失败。这是 fail-soft 设计，不影响本地文件处理流程。

6. **SQLite 外部数据源**：`database` source 类型需要提供服务器本地路径，前端有文字说明。

7. **时间预算（time_budget_seconds）**：后端接收此参数并传入 options，pipeline 中用于 LLM 超时降级判断，但不保证硬性截断（完整实现需改造 LLM 调用层）。

## 如何运行

```bash
# 启动服务
cd /home/match/LLM-DocFusion
./start.sh
# 或
source venv/bin/activate
python3 -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000

# 验证
curl http://localhost:8000/api/health

# 运行 smoke test（需服务已启动）
source venv/bin/activate
python3 scripts/smoke_api.py

# 运行单元测试
cd backend
source ../venv/bin/activate
python3 -m pytest tests -q
```
