# LLM-DocFusion Demo 运行说明（可运行 Demo 实现程序）

## 一、Demo 概述

本文档提供 LLM-DocFusion 系统的完整可运行 Demo 说明，包括：
- 系统环境要求与安装步骤
- 一键启动方式
- 多种使用方式（Web 界面 / API / 命令行脚本）
- 内置测试场景的完整运行步骤
- 常见问题排查

## 二、环境要求

### 2.1 硬件要求

| 配置项 | 最低要求 | 推荐配置 |
|:------:|:--------:|:-------:|
| 操作系统 | Linux（Ubuntu 20.04+）/ macOS 12+ | Ubuntu 22.04 LTS |
| CPU | 4 核 | 8 核以上 |
| 内存 | 16 GB RAM | 32 GB RAM |
| 显卡 | 可选（CPU 可推理） | NVIDIA GPU，≥16 GB VRAM（显著加速） |
| 磁盘空间 | 20 GB（含模型文件） | 50 GB SSD |
| 网络 | 首次安装需联网下载模型 | — |

> **提示**：仅使用 CPU 推理时，qwen2.5:14b 模型单次调用约需 10～30 秒（依 CPU 性能），整体任务约 30～90 秒。GPU 模式下约快 5～10 倍。

### 2.2 软件要求

| 软件 | 版本 | 安装方式 |
|:----:|:----:|:--------:|
| Python | 3.10 或 3.11 | 系统包管理器 / pyenv |
| Ollama | 0.3.0+ | 见 2.3 节 |
| Git | 2.0+ | 系统包管理器 |

## 三、安装步骤

### 3.1 安装 Ollama 并加载模型

```bash
# 安装 Ollama（Linux）
curl -fsSL https://ollama.com/install.sh | sh

# 启动 Ollama 服务
ollama serve &

# 下载并加载 qwen2.5:14b 模型（约 9 GB，需等待下载完成）
ollama pull qwen2.5:14b

# 验证模型已加载
ollama list
# 输出示例：
# NAME              ID            SIZE   MODIFIED
# qwen2.5:14b       ...           9.0 GB  ...
```

### 3.2 获取项目文件

```bash
# 如通过压缩包提交，解压到目标目录
unzip LLM-DocFusion.zip -d /opt/
cd /opt/LLM-DocFusion

# 或克隆仓库（如有）
git clone <repository_url> LLM-DocFusion
cd LLM-DocFusion
```

### 3.3 验证目录结构

```
LLM-DocFusion/
├── start.sh              ← 一键启动脚本
├── backend/
│   ├── requirements.txt  ← Python 依赖列表
│   └── app/              ← 后端源码
├── frontend/             ← 前端静态文件
├── config/
│   └── config.json       ← 系统配置
├── 测试集/               ← 内置测试数据
└── docs/                 ← 项目文档
```

## 四、启动方式

### 方式一：一键启动（推荐）

```bash
cd /path/to/LLM-DocFusion

# 添加执行权限（首次）
chmod +x start.sh

# 启动系统
./start.sh
```

**启动过程说明**：
1. 脚本自动检测 Python 版本
2. 自动创建 Python 虚拟环境（`venv/`）
3. 自动安装所有依赖（来自 `backend/requirements.txt`）
4. 自动选择可用端口（默认 8000，如被占用自动递增）
5. 启动 FastAPI 服务器

**预期输出**：
```
================================================
  DocFusion - 文档理解与多源数据融合系统
================================================
[1/3] 创建虚拟环境...
[2/3] 安装依赖...
[3/3] 启动服务器...

✅ 服务启动成功！
   前端界面：http://localhost:8000
   API 文档：http://localhost:8000/docs
   健康检查：http://localhost:8000/api/health
```

**自定义端口**：
```bash
PORT=8080 ./start.sh
```

### 方式二：手动启动

```bash
cd /path/to/LLM-DocFusion

# 创建并激活虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r backend/requirements.txt

# 创建必要目录
mkdir -p uploads outputs logs

# 启动服务器
python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --reload
```

## 五、Demo 测试场景运行步骤

### 场景一：通过 Web 界面处理（最直观）

#### 步骤 1：打开浏览器

访问 `http://localhost:8000`（或 `./start.sh` 输出的实际地址）。

#### 步骤 2：上传测试文件

推荐使用以下内置测试文件进行演示：

**数据源文件**（从以下任选一个或多个）：
```
测试集/包含模板文件/2025年中国城市经济百强全景报告/2025年中国城市经济百强全景报告.docx
测试集/包含模板文件/COVID-19数据集/COVID-19全球数据集（节选）.xlsx
测试集/包含模板文件/COVID-19数据集/中国COVID-19新冠疫情情况.docx
```

**模板文件**：
```
测试集/包含模板文件/2025年中国城市经济百强全景报告/2025年中国城市经济百强全景报告-模板.xlsx
测试集/包含模板文件/COVID-19数据集/COVID-19 模板.xlsx
```

在界面的"数据源文件"区域点击或拖入上述数据源文件，在"模板文件"区域上传对应模板。

#### 步骤 3：开始处理

点击"开始处理"按钮，观察实时进度条。处理阶段依次为：
- 解析文档（Parsing Documents）
- 解析模板（Parsing Template）
- 理解需求（Processing Requirement）
- 检索证据（Retrieving Evidence）
- 抽取数据（Extracting Data）
- 填充模板（Filling Template）
- 验证结果（Validating Result）

#### 步骤 4：查看并下载结果

处理完成后，页面显示：
- 填充率（如：城市百强样例 89.3%，COVID 样例 96.5%）
- 填充行数（如：100 行 / 14560 行）
- 处理耗时（如：74～86 秒）
- 置信度分布（高/中/低）

点击"下载结果文件"获取填充完成的文件。

### 场景二：通过 API 直接调用

#### 2.1 健康检查

```bash
curl -s http://localhost:8000/api/health | python3 -m json.tool
```

预期输出：
```json
{
  "status": "ok",
  "provider": "ollama",
  "ollama_status": "ok",
  "model": "qwen2.5:14b",
  "version": "1.0.0"
}
```

#### 2.2 本地文件直接处理（适合批量化场景）

```bash
curl -s -X POST http://localhost:8000/api/process/local \
  -H "Content-Type: application/json" \
  -d '{
    "source_files": [
      "测试集/包含模板文件/2025年中国城市经济百强全景报告/2025年中国城市经济百强全景报告.docx"
    ],
    "template_files": [
      "测试集/包含模板文件/2025年中国城市经济百强全景报告/2025年中国城市经济百强全景报告-模板.xlsx"
    ],
    "requirement": "",
    "options": {
      "use_llm": true
    }
  }' | python3 -m json.tool
```

#### 2.3 上传文件并处理

```bash
# 上传数据源文件
curl -s -F "files=@测试集/包含模板文件/COVID-19数据集/COVID-19全球数据集（节选）.xlsx" \
  http://localhost:8000/api/files/upload

# 上传模板文件
curl -s -F "files=@测试集/包含模板文件/COVID-19数据集/COVID-19 模板.xlsx" \
  http://localhost:8000/api/templates/upload

# 完整处理（上传+处理一体）
curl -s -X POST http://localhost:8000/api/process \
  -F "source_files=@测试集/包含模板文件/COVID-19数据集/COVID-19全球数据集（节选）.xlsx" \
  -F "template_files=@测试集/包含模板文件/COVID-19数据集/COVID-19 模板.xlsx" \
  -F "requirement=" \
  -F "use_llm=true" \
  | python3 -m json.tool
```

#### 2.4 查询任务状态（异步场景）

```bash
# 获取 task_id 后查询状态
curl -s http://localhost:8000/api/status/{task_id} | python3 -m json.tool

# 获取最终结果
curl -s http://localhost:8000/api/result/{task_id} | python3 -m json.tool
```

#### 2.5 下载输出文件

```bash
# 列出所有输出文件
curl -s http://localhost:8000/api/outputs

# 下载指定文件
curl -O http://localhost:8000/api/download/{output_filename}
```

## 八、配置修改说明

编辑 `config/config.json` 可调整以下参数：

```json
{
    "llm": {
        "ollama": {
            "api_base": "http://localhost:11434",
            "model_name": "qwen2.5:14b",
            "temperature": 0.3
        }
    }
}
```

| 参数 | 说明 | 建议值 |
|:----:|:----:|:------:|
| `api_base` | Ollama 服务地址（如部署在另一台机器上可修改） | `http://localhost:11434` |
| `model_name` | 使用的模型名称（可替换为其他已加载的模型） | `qwen2.5:14b` |
| `temperature` | LLM 输出随机性（越低越稳定） | `0.1`～`0.5` |

## 九、常见问题排查

| 问题现象 | 可能原因 | 解决方案 |
|:--------:|:--------:|:--------:|
| 启动报错"端口被占用" | 8000 端口已被其他程序使用 | `PORT=8080 ./start.sh` |
| 健康检查返回 `ollama_status: disconnected` | Ollama 服务未启动 | 执行 `ollama serve &` |
| 健康检查返回 `model: not found` | qwen2.5:14b 未加载 | 执行 `ollama pull qwen2.5:14b` |
| 文件上传报"格式不支持" | 文件扩展名不在白名单内 | 确认文件格式为 xlsx/docx/md/txt/csv |
| 处理很慢（>5分钟） | CPU 模式推理，机器性能有限 | 耐心等待，或使用 GPU 加速 |
| LLM 输出 JSON 解析失败 | 模型输出格式异常 | 系统会自动重试，或检查日志 `logs/` |
| `pip install` 失败 | 网络问题或 Python 版本不兼容 | 使用国内镜像：`pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r backend/requirements.txt` |
