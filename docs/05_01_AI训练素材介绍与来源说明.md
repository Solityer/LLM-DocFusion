# AI 训练素材介绍与来源说明

## 一、说明概述

LLM-DocFusion 系统的人工智能能力主要依赖于**预训练大语言模型**（qwen2.5:14b），本项目**未对模型进行任何自定义训练、微调（Fine-tuning）或增量预训练**。系统以工程化方式集成并调用现有预训练模型，通过精心设计的 Prompt 工程和混合抽取策略实现业务目标。

以下内容将从三个维度说明本项目涉及的 AI 素材情况：
1. 所使用的预训练模型及其训练数据来源声明；
2. Prompt（提示词）素材的设计方法与来源；
3. 系统测试集数据的来源与合规性说明。

## 三、Prompt 素材设计说明

### 3.1 Prompt 工程的角色

本项目的核心创新之一是**针对文档数据抽取场景设计的 Prompt 工程**。Prompt（提示词）是引导大语言模型完成特定任务的自然语言指令，属于工程实现层面的创新，而非训练素材。

### 3.2 主要 Prompt 类型

#### 类型一：结构化数据抽取 Prompt

**用途**：引导 LLM 从文本段落或表格中提取指定字段的数值/文本数据，并以 JSON 格式返回。

**设计原则**：
- 角色设定（Role-playing）：明确 LLM 扮演"专业数据提取助手"角色
- 格式约束（Format Constraint）：严格规定输出为 JSON 数组格式
- 示例引导（Few-shot Prompting）：提供完整输出示例降低幻觉率
- 低温度设置（Temperature=0.3）：减少随机性，提高一致性

**Prompt 模板示例**：
```
你是一个专业的数据提取助手，擅长从政府报告和统计文件中准确提取结构化数据。

【任务】
请从下方文本中提取数据，填充到以下表格列：
{headers_list}

【文本内容】
{text_content}

【输出要求】
1. 以 JSON 数组格式返回，每个元素代表一行数据
2. 键名必须与表格列名完全一致
3. 若某列数据不存在，对应值为 null
4. 只输出 JSON，不要添加任何解释

【输出示例】
[
  {{"列名1": "值1", "列名2": "值2"}},
  {{"列名1": "值3", "列名2": "值4"}}
]
```

**来源说明**：Prompt 模板由项目团队原创设计，参考了以下公开学术/工程资源：
- Wei et al. (2022), *Chain-of-Thought Prompting*（思维链提示方法论）
- OpenAI Cookbook 中的 JSON 输出控制方法（公开工程文档）
- few-shot prompting 最佳实践（学术界公开研究成果）

#### 类型二：需求语义解析 Prompt

**用途**：将用户自然语言需求解析为结构化的 RequirementSpec（时间范围、关键词、过滤条件）。

**示例**：
```
请分析以下用户需求，提取关键信息：

【用户需求】
{requirement_text}

请以 JSON 格式返回：
{{
  "time_range": {{"start": "...", "end": "..."}},
  "entity_keywords": ["关键词1", "关键词2"],
  "filter_conditions": {{"条件名": "条件值"}},
  "focus_fields": ["字段1", "字段2"]
}}
```

#### 类型三：字段语义对齐 Prompt

**用途**：判断源文档的字段名称是否与模板表头语义等价（处理同义词、缩写、全称变体）。

**示例**：
```
判断以下两个字段名称是否指代相同的数据指标：
字段A：{field_a}
字段B：{field_b}

背景：这是一份{context}类型的报告。
请回答 "是" 或 "否"，并简要说明理由。
```
