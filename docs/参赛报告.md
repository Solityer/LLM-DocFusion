# DocFusion 参赛报告

**项目名称**：DocFusion — 多源数据整合与质量识别应用系统  
**版本**：v2.0  
**技术栈**：Python 3.12 · FastAPI · SQLite3 · Ollama + qwen2.5:14b · Vanilla JS

---

## 一、项目概述

DocFusion 是一个基于大语言模型的文档智能处理平台，解决"多源异构数据→标准化模板填充"的全流程问题。系统覆盖数据接入、质量识别、LLM 增强抽取、模板填充、结果评估等完整链路，并将所有处理过程写入本地 SQLite 资产库，支持全文检索与离线分析。

---

## 二、系统架构

```
用户浏览器（单页应用）
        │
        ▼
FastAPI 后端（port 8000）
   ├── /api/process/*       主填表流程
   ├── /api/sources/*       多源数据接入与预览
   ├── /api/document/*      文档智能操作
   ├── /api/store/*         SQLite 文档资产库
   ├── /api/templates/*     模板检视
   ├── /api/analytics/*     分析看板
   └── /api/evaluate/*      竞赛评估
        │
        ├── Ollama (qwen2.5:14b)   LLM 推理
        └── SQLite (docfusion_store.sqlite)   持久化存储
```

### 数据流（主填表）

```
上传文件 → 文档解析(DocumentBundle) → 模板解析(TemplateSchema)
    → 需求解析(RequirementSpec) → 证据检索(RetrievalResult)
    → 数据抽取(规则+LLM混合) → 多源融合 → 模板填充
    → 质量验证 → 输出生成 → [可选] SQLite 入库
```

## 三、核心功能实现

### 3.1 多源数据整合

系统支持 4 种数据源类型，通过统一的 `SourceSpec` 接口接入：

| 类型 | 实现模块 | 说明 |
|------|---------|------|
| 本地文件 | `document_service.py` | xlsx/docx/csv/txt/md/json |
| HTTP/API 接口 | `api_source_service.py` | JSON/CSV 响应自动解析 |
| 网页抓取 | `web_source_service.py` | HTML → 文本/表格提取 |
| SQLite 数据库 | `database_source_service.py` | 任意 SELECT 查询 |

多源数据通过 `fusion_service.py` 按字段置信度加权融合，避免单源偏差。

### 3.2 数据质量识别

`quality_service.analyze_documents()` 检测 5 类质量问题：

- **缺失值**（`missing_value`）：必填字段为空
- **异常值**（`anomaly`）：超出统计范围的数值
- **重复值**（`duplicate_value`）：相同实体多行重复
- **格式不一致**（`format_inconsistency`）：日期/数值格式混杂
- **单位不一致**（`unit_inconsistency`）：亿元/万元混用

所有问题写入 `quality_issues` 表，并在处理结果中输出结构化报告。

### 3.3 LLM 集成与混合策略

- **模型**：Ollama 本地部署 qwen2.5:14b，默认 temperature=0.1
- **混合策略**：规则路径优先，LLM 仅处理规则无法覆盖的字段
- **降级保障**：Ollama 不可用时自动切换规则路径，填充率轻微下降但系统不崩溃
- **用量统计**：每次任务记录 LLM 调用次数、Token 用量，展示在前端

### 3.4 文档智能操作

`document_operation_service.py` 支持 7 种自然语言驱动的文档操作：

```
summarize / extract / find / replace / export_table / format / inspect
```

操作类型可自动识别（通过关键词匹配），也可手动指定。

### 3.5 SQLite 文档资产库

8 张表的完整 schema：

```sql
documents         -- 文档元数据（SHA256 去重）
text_blocks       -- 文本块（段落级）
tables            -- 表格结构（headers, row_count）
table_rows        -- 表格行数据（原始值 + 标准化值）
extracted_entities -- 抽取实体（城市/机构，规则提取）
extracted_fields  -- 抽取字段（含标准化值和字段类型）
quality_issues    -- 质量问题记录
task_metrics      -- 任务指标（填充率/响应时间/质量问题数）
```

**关键设计**：
- WAL 模式支持并发读写
- SHA256 哈希去重，避免重复入库
- `overwrite=True` 参数允许强制覆盖
- 背景线程异步入库，不阻塞主请求

### 3.6 模板字段增强

新增 `options` 字段支持：

```json
{
  "use_llm": true,
  "strict_mode": false,
  "enable_quality_detection": true,
  "enable_data_fusion": true,
  "store_extracted_data": false,
  "time_budget_seconds": 90,
  "field_aliases": {
    "GDP总量（亿元）": ["地区生产总值", "GDP"],
    "城市名": ["城市", "地区"]
  }
}
```

模板检视 API（`/api/templates/inspect/upload`）返回 `TemplateSchema`，包含所有表格 headers、placeholders 和字段列表。

## 六、创新点

1. **证据链透明**：每个填充字段均记录来源文件、位置、原始片段和置信度，支持追溯验证
2. **实体语义合并**：`_merge_records_by_semantic_key()` 将"南京"与"南京市"识别为同一实体，跨源合并数据
3. **跨主题隔离**：`_source_matches_template_context()` 防止不相关数据源污染填充结果
4. **粒度感知实体过滤**：`evaluate_entity_compatibility()` 区分国家级/省级/市级粒度，避免错误回填
5. **混合提取策略**：规则路径覆盖结构化表格（速度快），LLM 路径覆盖非结构化叙述文本（准确率高）

## 七、部署说明

```bash
# 前置：安装 Ollama 并拉取模型
ollama pull qwen2.5:14b

# 启动服务（自动创建 venv + 安装依赖）
bash start.sh

# 访问
http://localhost:8000
```

数据库自动创建于 `data/docfusion_store.sqlite`，无需手动初始化。
