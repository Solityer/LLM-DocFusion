# DocFusion 演示脚本

本文档为评审/演示准备的完整操作流程，覆盖所有核心功能模块，预计演示时间约 15–20 分钟。

---

## 前置条件

1. 启动服务：`cd /home/match/LLM-DocFusion && bash start.sh`
2. 打开浏览器：`http://localhost:8000`
3. 确认右上角模型状态显示绿色"在线"

---

## 一、主填表流程演示（约 5 分钟）

### 步骤 1：上传数据源文件

1. 在"📁 文档上传与主填表流程"区域，将以下文件拖拽至左侧"数据源文件"上传区：
   - `测试集/包含模板文件/2025年中国城市经济百强全景报告/2025年中国城市经济百强全景报告.docx`
2. 文件列表中出现文件名和大小，表示上传成功

### 步骤 2：上传模板文件

1. 将以下文件拖拽至右侧"模板文件"上传区：
   - `测试集/包含模板文件/2025年中国城市经济百强全景报告/2025年中国城市经济百强全景报告-模板.xlsx`
2. 点击"🔍 预览模板字段"，观察弹出的字段列表（城市名、GDP总量、常住人口等）

### 步骤 3：配置处理选项

1. 在"📝 用户要求与处理选项"中，填写需求（可留空让系统自动识别）：
   ```
   将城市经济数据填入模板中
   ```
2. 确认勾选：LLM 增强、质量检测、多源融合
3. 时间预算设为 90 秒

### 步骤 4：开始处理

1. 点击"🚀 开始处理"
2. 观察"⚙️ 处理流程"区域的 9 个步骤依次变绿：
   - 环境清理 → 文档解析 → 模板解析 → 需求解析 → 证据检索 → 数据抽取 → 模板填充 → 结果验证 → 输出生成
3. 等待进度条到达 100%（约 30–90 秒）

### 步骤 5：查看结果

1. 在"📊 处理结果"区域，观察：
   - 填充率（应 ≥ 80%）
   - 字段置信度表格
   - 数据源贡献统计
2. 点击"下载"按钮下载填写完成的 Excel 文件

---

## 二、多源数据接入演示（约 3 分钟）

### 步骤 1：添加 HTTP/API 数据源

1. 在"🌐 多源数据接入"区域，选择来源类型"HTTP/API 接口"
2. 填写名称："示例 API 数据"
3. 填写 URL（可使用本地测试端点或真实 API）
4. 点击"👁 预览数据源"，观察返回的数据行数和字段列表
5. 点击"➕ 加入处理"，观察上方数据源列表新增一条记录

### 步骤 2：添加本地文件数据源

1. 保持默认类型"HTTP/API 接口"，改选"网页抓取"
2. 填写一个公开 URL
3. 点击"预览数据源"，确认可获取内容

### 步骤 3：添加 SQLite 数据库数据源

1. 选择类型"SQLite 数据库"
2. 填写数据库路径：`data/docfusion_store.sqlite`
3. 填写 SQL：`SELECT * FROM documents LIMIT 10`
4. 点击预览，观察返回的文档列表

---

## 三、文档智能操作演示（约 3 分钟）

### 步骤 1：文档摘要

1. 在"🤖 文档智能操作"区域，上传任意文档（Word/TXT/Excel）
2. 操作类型选"摘要（summarize）"
3. 指令填写：`总结这份文档的核心内容，提取关键指标`
4. 点击"▶ 执行操作"，观察摘要结果

### 步骤 2：字段提取

1. 重新上传一份包含表格的 Excel 文件
2. 操作类型选"字段提取（extract）"
3. 指令填写：`提取文档中的城市、GDP、人口字段`
4. 观察返回的结构化 JSON 数据

### 步骤 3：文本替换

1. 上传一份包含占位符的 Word 文档
2. 操作类型选"替换（replace）"
3. 指令填写：`把 {{公司名称}} 替换为 北京示例科技有限公司`
4. 观察替换后的文档内容预览

## 四、非结构化文档入库演示（约 2 分钟）

### 步骤 1：文档入库

1. 在"🗄️ 非结构化文档数据库存储"区域，拖拽上传 2–3 份文档（TXT/Word/CSV）
2. 确认勾选"实体提取"
3. 点击"📥 执行入库"
4. 观察入库进度（状态栏显示：读取 → 质量分析 → 写入数据库）

### 步骤 2：查看已入库文档

1. 点击"📋 查看已入库"，观察文档列表（文档 ID、文件类型、文本块数、表格数）

### 步骤 3：全文搜索

1. 在搜索框输入关键词（如"GDP"、"城市名"、"人口"）
2. 点击"🔍 搜索"，观察搜索结果（按类型分组：文档/文本块/实体/字段）

## 常见问题

**Q: 模型状态显示"离线"怎么办？**
A: 执行 `ollama serve` 并确认 `ollama run qwen2.5:14b` 可正常响应。处理时关闭 LLM 开关仍可使用规则路径。

**Q: 处理超时怎么办？**
A: 适当增加"时间预算"秒数，或减少同时处理的模板数量。

**Q: 数据库在哪里？**
A: `data/docfusion_store.sqlite`，可用 SQLite Browser 直接查看 8 张表的数据。
